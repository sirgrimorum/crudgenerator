# Changelog

All notable changes to CrudGenerator will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [3.8.38] - 2026-03-04

### Fixed
- `lists()` now passes the raw filtered Eloquent collection to the list blade instead of pre-computed celda-array rows. The list view was designed for Eloquent model iteration; passing celda arrays caused "Array to string conversion" errors in PHP 8.2 when any records were present

## [3.8.37] - 2026-03-04

### Fixed
- Added missing `validateOlderThan()` and `replaceOlderThan()` methods to `ExtendedValidator` that were accidentally omitted in 3.8.36
- Fixed "Array to string conversion" when `$config['nombre']` maps to an array in the list view `<tr id>` attribute

## [3.8.36] - 2026-03-04

### Fixed
- `ExtendedValidator` constructor: changed `$this->messages[...]` to `$this->customMessages[...]` to match the Laravel 9+ validator API (the `$messages` property is no longer publicly writable)
- `CrudStrings::isFunctionOfType()` and `isFunction()`: added type guard (`!is_object && !is_string → return false`) to prevent `TypeError` when PHP 8.2 receives a non-object from array-access
- `list.blade.php`: replaced `$value->{$property}` property access with `data_get($value, $property)` throughout — prevents `TypeError: Attempt to read property on array` in PHP 8.2
- `list.blade.php`: `number_format()` calls now guard against array inputs with an explicit cast to prevent `TypeError` on PHP 8.2

### Added
- `CrudGenerator::shouldShowButton($config, $buttonName)` — checks whether a button should be rendered for a given config
- `CrudGenerator::addScriptLoaderHtml()` — returns the script loader HTML tag (configurable via `scriptLoader_name`)
- `CrudGenerator::addLinkTagLoaderHtml()` — returns the CSS link tag loader HTML (configurable via `linkTagLoader_name`)

## [3.8.35] - 2026-03-02

### Added
- Laravel 12 support
- Minimum PHP version raised to 8.2

### Changed
- Removed `laravelcollective/html` dependency; form HTML is now generated by the internal `FormHelper` class, eliminating the abandoned package dependency

### Fixed
- Removed dead `getDoctrineSchemaManager()` call in `CrudConfig::getModelDetailsFromDb()` — the method was removed in Laravel 11 and the variable was only used in commented-out code

## [3.8.34] - 2026-03-01

### Added
- Scrutinizer CI configuration using PHP 8.2 and PHPUnit 10

## [3.8.33] - 2026-03-01

### Fixed
- Replaced deprecated `Translator::trans()` call with `Translator::get()` in `CrudStrings::translateString()` (Laravel 9+)
- Guarded empty prefix in `translateString()` to avoid spurious lookups

## [3.8.32] - 2026-03-01

### Fixed
- Removed dead `DB::connection()->getDoctrineSchemaManager()` call and fixed `getJustValue()` return value

## [3.8.31] - 2026-03-01

### Added
- Comprehensive test suite (13 feature tests) using MySQL and orchestra/testbench

### Fixed
- Multiple Laravel 9+ compatibility issues discovered during test suite development

## [3.8.30] - 2026-02-26

### Changed
- Updated compatibility to PHP ^8.x and Laravel ^9.0|^10.0|^11.0
- Minimum PHP raised from 7.x to 8.x

## [3.8.29] - 2021-03-01

### Added
- Prevent error when a `file` or `files` field is empty during create/update

## [3.8.28] - 2021-02-26

### Fixed
- Error with extra arguments in `transstring()` method

## [3.8.27] - 2021-02-xx

### Fixed
- Various DataTables pre-filter cookie handling improvements

## [3.8.26] - 2021-02-xx

### Fixed
- Server-side DataTables edge cases

## [3.8.25] - 2021-02-xx

### Added
- `conditions` toggle button support in DataTables

## [3.8.24] - 2021-02-xx

### Added
- `filters` toggle button support in DataTables

## [3.8.23] - 2021-02-xx

### Added
- Pre-filter persistence via cookies (5-minute default TTL)

## [3.8.22] - 2021-xx-xx

### Added
- Server-side DataTables processing option (`serverSide: true`)

## [3.8.21] - 2021-xx-xx

### Added
- DataTables `multiRemove` support for batch deletion via multi-select

## [3.8.20] - 2021-xx-xx

### Added
- `ajax` option for DataTables integration (client-side data loading)
